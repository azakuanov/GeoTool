Versions

This is a list of relevant recent changes per version.

v1.8.6

Updated FourSquare credentials.

v1.8.5

I've made some minor improvements that make GeoLicious more stable and reliable.

I'm happy for (nearly) every review in the AppStore and recommendations to new users. GeoLicious is and will be a hobby and every cent returned motivates me to keep working on it. Thank you.

v1.8.4

There have been some minor optimizations and tweaks.

  • The loading screen is now properly displayed on the iPad too.
  • The summary below all lists is back again.
  • Handling of automated checkins has been further improved.
  • Some things have been optimized for better performance.

v1.8.3

Bugs come and go... unfortunately the app forgot its location if automated check-ins had been disabled. This has been corrected and you may now check in again as used to before.

v1.8.2

This version fixes some minor bugs and improves reliability.

Furthermore I've successfully removed the limit of 20 locations for automated checkins. From now on you may activate an unlimited number of locations if they are not too close to each other.

v1.8.1

With the introduction of iCloud Drive with the last update I've noticed some obvious improvements... here they are.

  • All settings are now stored locally on your device and no longer shared via iCloud synchronization.
  • The overall reliability of automated checkins for large databases has been improved by several performance optimizations.
  • Manual backups are now labeled with a timestamp rather than an incrementing number.
  • The automated check-out of previously entered locations is now performed for the last 5 if possible.
  • Fixed some minor bugs related to iCloud Drive updates.

I'm happy for (nearly) every review in the AppStore and recommendations to new users. GeoLicious is and will be a hobby and every cent returned motivates me to keep working on it. Thank you.

v1.8

Finally you have the option to store your data in iCloud Drive! This includes the normal database as well as automated and manual backups. This way your data will be automatically stored in the cloud and sync to all your iOS devices. Additionally you will be able to access and save that data on your Mac or via www.iCloud.com. And of course all that is optional.

Furthermore some little things improved...

  • If you've purchased the Pro Version as InApp-Purchase, you can now more easily restore that feature using the settings.
  • The list of checkins now names the weekday if you've grouped your entries by day.
  • I've fixed some fonts in some dialogs.

v1.7.3

I've fixed another bug related to checkins from different timezones on the same day.

v1.7.2

There have been some new bugs that I hope to fix with that update.

  • Using satelite mode for the map should no longer crash GeoLicious.
  • Authenticating the app to FourSquare should work again.

v1.7.1

I've made some small changes that make GeoLicious an even better app:

  • Notifications now come with two new options: Discard and CheckOut – right at the lockscreen.
  • The icon selector now includes all known icons - even those from Foursquare venues.
  • The small indicator icons at checkins now update visibly without scrolling.

v1.7

With this version I've switched GeoLicious to an InApp-Puchase for the full version. All former purchases will be upgraded automatically – even if you reinstall on any other iOS device. Who ever downloads GeoLicious for the first time from now on will have 10 free entries until the upgrade to the full version is needed.

I've also tried to add some more features. Here is a complete list.

  • Locations can now get an additional comment too.
  • A search for geofence now shows all locations with currently activated auto checkin.

Further more I still work on support of iCloud for easy sync between your devices and an automated backup of your data. Because of the complexity of changes and related tests I still cannot predict when I'll finally make these changes available.

v1.6.3

This version includes some preparations for the next major update and tries to fix another minor bug.

v1.6.2

Fixed another bug that caused random crashes with new checkins while the map was still updating.

v1.6.1

I've fixed some rare bugs.

v1.6

It has been time for a bigger update of GeoLicious! With the introduction of iOS8 some things have changed. I've also implemented some new features, that hopefully are as useful to you as they are to me.

  • Exported databases can now be loaded directly from Email or DropBox into GeoLicious. This allows you to easily share your data with other devices.
  • The app now remembers the last visible view and restores it upon a fresh start.
  • The map now zooms in to your last visible region with a fresh start of the app – if you wish so.
  • New locations via Foursquare now default to your current social settings. Known locations rely on their own settings.
  • Foursquare icons now will show up properly if you restore a database to a device that did not previously download them.
  • GeoLicious has been prepared for iPhone 6 and iPhone 6 Plus.

For v1.7 I plan the switch to iCloud and therefor real synchronization between different iOS devices. Furthermore I work on waypoints, which connect different checkins on the map and may be placed automatically if desired.

v1.5.4

One more try with another random crash on new checkins... the slower iPhone 4 should be stable now.

v1.5.3

A simple double-tap now toggles the date picker mode between date and time. That makes it much easier to add entries several years in the past.

v1.5.2

The random crash on new checkins has been resolved – finally!

v1.5.1

The last version introduced a small bug where newly added checkins didn't show up as expected – which hopefully has been fixed now. I've also managed to improve the overall performance of large databases.

v1.5

From now on GeoLicious is available as universal app for iPhone and iPad. To benefit from all features your iPad should have an integrated SIM slot and GPS functionality. Otherwise location specific features will be limited in their accuracy.

I'm no longer able to maintain iOS6 support due to missing hardware and test capabilities. So I've decided to stick to iOS7 only. The upcomming iOS8 will be supported too – that's sure.s

The following small changes have been made too:

  • The app icon has been slightly improved with some additional details.
  • Editing the date of some checkin now uses an inline date picker.
  • You may now search the list of checkins for dates like 2013, 10/2011 or 04/25/2010.
  • On the iPad the map zooms in to a location, if you select a checkin from one of the lists.
  • I've removed landscape mode for the moment – will be added later with new features.
  • The footer counting the checkins and locations now counts what is shown.
  • Some minor bugs have been fixed.

v1.4.2

GeoLicious should now feel a little faster with large databases.

v1.4.1

I've tried to fix a crash some users reported with checkins to new locations – let me know if it fixes this one for you.

v1.4

There is now a free version called GeoLicious Free you may recommend to those friends who are not sure about the app and willing to test it without risk. The free version allows up to 10 entries and comes with the full functionality of the full version.

Your data can now be printed using an AirPrint enabled printer. Simply filter all your checkins or locations, group as you prefer and start printing. When you wish to print your checkins you may also choose from some predefined timeranges.

The following changes have been made to the social networking section:

  • Now it is possible to switch Foursquare, Facebook and Twitter on or off by using the detailed checkin via the accessory button. This will also respect the comment and use it for Foursquare.
  • Every location now remembers not only the Foursquare setting but also whether to link Facebook and Twitter.
  • In addition to the active and inactive Foursquare icon you'll now see Facebook or Twitter icons for all checkins and locations, if these options are active.
  • A small bug with rare double checkins on Foursquare has been squashed.

Furthermore there are these minor additions:

  • GeoLicious now notifies you of inefficiant settings during app start or when activating related features. This includes disabled location services or missing background modes.

At the end of list of all checkins you'll now find a short summary of your data.

You may now sort the list of checkins in reversed order – new entries will be on top.

v1.3.2

You may now choose one of many different sounds for checkin notifications or turn them off completely – right within the app.

v1.3.1

If you use the new Foursquare sync you may now also sync these checkins to Facebook or Twitter if you've connected these networks with Foursquare. Otherwise the setting won't have any effect. A direct connection to these networks is not supported yet.

The link to @GeoLiciousApp now prefers Tweetbot over Twitter over plain web.

v1.3

This version of GeoLicious intruduces an active connection to Foursquare. Those who like may now sync selected locations to the social network. This works for manual checkins as well as automated checkins in the background.

Of course this functionality is fully optional – you may disconnect Foursquare at every time and reconnect later it desired.

The following minor changes have been made additionally.

  • You may connect existing locations to Foursquare for later synchronisation. This will also update name and icon.
  • The short alert with time since last checkin now respects checkout time if available.
  • The icon scroller for locations no longer moves vertically.
  • GeoLicious now shows a small alert with the first start after an update.
